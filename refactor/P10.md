## Scope

### 1. Saved-state cleanup

`crates/core/src/state.rs::ResolvedSettings` and the surrounding `State`
JSON loader survived P5.d's "all config in `init.lua`." That's a doc
lie — config is Lua, but persisted user state still loads through a
typed JSON shim. Two paths forward; pick during P10:

- **A.** Drop persisted `ResolvedSettings` entirely. State that the user
  controls via Lua does not need to round-trip through a settings
  registry; what survives is genuinely *cache-shaped* (last-model,
  branch, recent slugs). Move the cache half to a small typed
  `SessionCache` and delete the rest.
- **B.** Keep persisted state as a Lua-driven save: `smelt.session.save_meta(t)`
  / `smelt.session.load_meta()` that reads/writes a JSON blob owned by
  Lua. Removes the Rust schema; preserves persistence.

Either way, `state.rs` shrinks ≥ 75% and the doc claim "all config in
init.lua" is true at the file level.
