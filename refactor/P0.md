P0 originally planned 9 deletions. Five landed cleanly with the tree
green throughout. Four were structural — call sites had nothing to
point at without their P1 replacements existing — and moved to P1.0
to be paired with their replacements in the same commit.

  `NotificationStyle`, `DialogConfig`, `DrawContext`, `Compositor`, plus
  `Ui::set_selection_bg` / `Ui::selection_style()` shim.

  `classify_widget_action`.
- [x] `MouseAction::Yank(String)` and `WidgetEvent::Yank(String)`.
- [x] `crates/ui/src/buffer_list.rs`.

Deferred to P1.0 (paired with their replacements):

- [ ] `crates/ui/src/buffer_view.rs` + every `BufferView::new(...)` call
  site. Pairs with `Window::render(buf, grid)` (P1.d).

  Pairs with Window as the only interactive unit (P1.d).
- [ ] `PanelWidget` trait + panel-widget multiplexing in `dialog.rs`.
  Pairs with the "Overlay + LayoutTree + N Windows" rebuild (P1.c).
- [ ] 6-variant `Placement` enum + `add_layer` / `register_split` /

End state: tree green. `cargo nextest run --workspace` passes 901

## Decisions made while coding

- **Strategy shift mid-phase**: original plan said "P0 ends red, P1
  rebuilds". In practice, four of the nine deletions couldn't land
  without their P1 replacements — call sites had nothing to point at,
  and the binary stopped building (so the 5 baseline scenarios couldn't
  run as a regression gate). Rather than burn the green-tree baseline
  for ideological purity, we pair each structural deletion with its
  replacement in P1.0. The orthogonal four (selection_style fields,
  mouse-with-lua, Yank variants, BufferList) shipped cleanly because
  their replacements either didn't exist yet (selection styling moves
  to `theme.get("Visual")`, host-side) or were trivial (yank goes
  direct to clipboard).
- **Notification selection state stripped entirely**: with
  `selection_style` gone, `Notification::handle_mouse` had no visible
  effect — selection painting was the only consumer of the anchor /
  cursor_col fields. Whole `handle_mouse` impl + selection state
  removed; selection on toasts is gone until P1 rebuilds it through the
  general selection path.
- **Lua "list" panel kind removed**: `BufferList`'s consumers in Lua
  (`/agents`, `/resume`) lose their list panels. `kind = "list"` now
  errors with "unknown panel kind: list". P4 rebuilds list-of-buffers
  views as N Windows in an Overlay.
- **Dead BufferList tests deleted, not migrated**: 6 tests in
  `dialog.rs` exercised general dialog behaviour using `BufferList` as
  one panel. None had been migrated to `OptionList` because the new
  architecture redesigns dialog routing entirely. P1.c rebuilds test
  coverage against the Overlay+Windows model.
