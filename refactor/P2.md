# P2 — TUI App restructure

**Status:** in-progress **Started:** 2026-04-30 **Landed:** —

P2 splits today's god-struct `App` into a headless-safe `Core` plus
`TuiApp` / `HeadlessApp` frontends, carves the subsystems out as
their own types (Session, Confirms, Clipboard, Timers, Cells,
LuaRuntime, ToolRuntime, EngineBridge, AppConfig, WellKnown), installs
the `Host` + `UiHost` traits (absorbing P1.e's trait + supporting
types), and collapses the event surface onto a single `select!` loop
publishing through `Cells`. Sub-phases land independently, each green
at the boundary. Detail per sub-phase lives in the commit message;
this file is an index.

## Sub-phases landed

- **P2.a.1** (`b07f803`) — `WellKnown { transcript, prompt, statusline }` carves the well-known `WinId`s off App; `Ui::win_buf{,_id,_mut}` resolves `WinId` → backing `Buffer`.
- **P2.a.2** (`03bd06e`) — `WellKnown.cmdline: Option<WinId>` absorbs today's `App::cmdline_win`.

## Decisions made

- **WellKnown shape** (`b07f803`) — three stable WinIds (prompt /
  transcript / statusline) only; cmdline (currently
  `App::cmdline_win: Option<WinId>`) folds in next sub-phase. The
  redundant `input_display_buf` / `transcript_display_buf` /
  `status_buf` `BufId` fields retire entirely — buffers are reached
  through the new `Ui::win_buf{,_id,_mut}` helpers, matching the
  ARCHITECTURE.md target where `WellKnown` carries only WinIds.
- **P2.a sub-phase split** (`b07f803`) — P2.a is too large for one
  session; split into `a.1` … `a.12` with independent landings.
  Order is data-only carve-outs first (WellKnown, Confirms,
  AppConfig, Session), then new reactive primitives (Cells, Timers),
  then bridges (LuaRuntime/Cells fold, ToolRuntime, EngineBridge),
  then the Core / TuiApp / HeadlessApp aggregate.
