# P2 — TUI App restructure

**Status:** done **Started:** 2026-04-30 **Landed:** 2026-04-30

P2 splits today's god-struct `App` into a headless-safe `Core` plus
`TuiApp` / `HeadlessApp` frontends, carves the subsystems out as
their own types (Session, Confirms, Clipboard, Timers, Cells,
LuaRuntime, ToolRuntime, EngineBridge, AppConfig, WellKnown), installs
the `Host` + `UiHost` traits (absorbing P1.e's trait + supporting
types), and collapses the event surface onto a single `select!` loop
publishing through `Cells`. Sub-phases land independently, each green
at the boundary.

## Decisions made

- **WellKnown shape** — three stable WinIds only; redundant `BufId` shadows retire, buffers reach via `Ui::win_buf{,_id,_mut}`.
- **P2.a sub-phase split** — split into a.1…a.12: data lifts first, reactive primitives next, bridges, then the Core / TuiApp / HeadlessApp aggregate.
- **P2.a.3 split into a.3a / a.3b** — data carve-out is mechanical; oneshot::Sender swap + `is_clear()` gate need Cells (a.4) and EngineBridge.
- **AppConfig holds runtime-mutable fields too** — `mode` / `reasoning` / `model` flip during a session and feed Cells migration; segregating "immutable" from "runtime-mutated" would split one logical unit.
- **Data-only carve-outs precede a.4 (Cells)** — REFACTOR's text body ("data lifts first") trumps its numerical listing; a.6 / a.7 / a.8 are pure data and can land before the primitive.
- **P2.a.7 split into a.7a / a.7b** — snapshots is small + contained; `history → session.messages` is ~25 rename sites, too large to combine.
- **`Session.session_cost_usd` is persisted** — `#[serde(default)]` field rather than computed from `cost_snapshots.last()`; load path stays correct without a schema migration.
- **Timers lands before Cells** — Timers is independent of Cells; landing first means `now` / `spinner_frame` drivers are ready when Cells arrives.
- **a.8 fits the data-only carve-out tier** — routing the two clipboard bypass paths through `app.clipboard.*` is mechanical, finishes the data tier ahead of a.4.
- **P2.a.4 splits into a.4a / a.4b / a.4c** — primitive + bindings (a.4a); userdata-handle ergonomics + glob/au alias (a.4b); built-in cell migrations + Rust subscribers (a.4c).
- **`Rc<dyn Any>` over a custom `CellValue` trait** — blanket-impl `as_any` produced wrong TypeId; `Rc<dyn Any>::downcast_ref` works directly with no trait dispatch.
- **Single Lua-only `SubscriberKind` variant for now** — Rust variant warned dead (no built-in subscriber yet); a.4c adds `Rust(...)` when the first Rust-typed subscriber lands.
- **`PendingCallback { kind, is_glob }` over a new SubscriberKind variant** — keeps the kind enum to one shape; the drain pump branches on `is_glob` to call `fn(value)` vs `fn(name, value)`.
- **Glob subs share the SubscriptionId space** — one `next_id` counter for direct + glob keeps the bookkeeping uniform.
- **Strict `smelt.au.*` aliasing — no auto-declare** — `au.on` / `au.fire` return `nil` / `false` for undeclared cells rather than auto-declaring with `nil`.
- **a.4c splits into a.4c.1 / a.4c.2 / a.4c.3** — machinery + simple-typed setters; timer-driven cells; event-shaped cells (some gated on EngineBridge a.11).
- **Per-`TypeId` `LuaProjector` over per-cell projectors** — projector keyed by `TypeId::of::<T>()` not by cell name. One `String` projector serves every `String`-typed cell; new types register once.
- **`EventStub` placeholder over deferred declaration** — declare every event-shaped cell at startup with an `EventStub` value so `smelt.au.on("turn_complete", fn)` succeeds today.
- **Diff-publish for `vim_mode` / `confirms_pending`** — `Cells::publish_if_changed` short-circuits on equal values; `App::publish_diff_cells` runs once per tick.
- **`now` is unix seconds u64, not a chrono `DateTime`** — no chrono / time crate is in the tree; publishing a u64 keeps the projector primitive.
- **`tokens_used` excludes background requests** — auxiliary requests carry `background = true` so a subscriber sees only the user-visible context flow.
- **`spinner_frame` rolls back to 0 when no turn is animating** — `publish_diff_cells` reads `working.is_animating()` and falls back to 0.
- **`EventStub` declarations stay; typed projectors route by stored TypeId** — keeping the placeholder declared at startup means `:get()` before first publish reads `nil`.
- **`ConfirmChoice` projects as a stable short string, not the Rust enum** — `decision_label` returns `"yes" | "no" | "always_session" | …`. Plugins branch on the label.
- **`confirm_requested` cell carries the full request snapshot** — ARCHITECTURE.md's earlier `Cell<HandleId>` sketch becomes `Cell<ConfirmRequested>`.
- **`is_clear()` over `is_empty()` for the engine-drain gate** — same predicate, but the canonical name signals intent at call sites.
- **mode_change / model_change autocmd events fold into `agent_mode` / `model` cells** — the cell is the only signal; subscribers wanting `from` track it locally.
- **turn_end stays a separate cell from turn_complete** — `turn_complete` only fires on natural end; `turn_end` fires on every termination.
- **smelt.on retires entirely** — migrating the two plugin callers (plan_mode, predict) is one-line each; no third-party plugins exist yet.
- **Engine-drain gate uses `confirms.is_clear()`, not `blocks_agent`** — the `blocks_agent` flag mixes "tool confirm pending" with "any modal UI absorbs input". Only the first is a real reason to pause engine streaming.
- **EngineBridge field stays named `engine` for now** — renaming to `engine_bridge` rides the `Core` aggregate (P2.a.12).
- **a.12 splits into a.12a / a.12b** — the subsystem aggregation (~350 callsite renames) and the frontend split are each natural single-session units.
- **`Core` lives behind `App.core: Core` for now, not the inverse** — wrapping with `App.core: Core` means the 8 subsystem fields move without touching the UI fields.
- **a.10 (ToolRuntime carve-out) deferred** — REFACTOR's "may defer (depends on P5.a's trait shape)" applies.
- **`pub` vs `pub(crate)` on Core fields mirrors the prior App layout** — `Confirms`, `Timers`, `Cells`, `EngineBridge` are `pub(crate)` types; their Core fields are `pub(crate)` to match.
- **a.12b splits into a.12b1 / a.12b2** — HeadlessApp carve (~600 LOC) and `App` → `TuiApp` rename (~300+ callsites) are each self-contained.
- **`Core::new(config, engine)` is the only Core construction path** — both `App::new` and `HeadlessApp::new` call it.
- **Headless skips saved-state seeding** — headless / subagent invocations must be predictable from the CLI alone.
- **`HeadlessSink::new_subagent` is JSON + verbose, no flags** — subagents always emit every engine event as a JSON line.
- **`build_headless_config` lives in `src/main.rs`, not in `tui::app`** — the helper is one caller's assembly.
- **Color resolution stays a process-wide static** — `COLOR_OVERRIDE: OnceLock<Option<bool>>` and the `stderr_supports_color` cache live at module scope.
- **P2.b splits into b.1 / b.2 / b.3 / b.4 / b.5** — five natural single-session units: Host trait, UiHost trait, supporting types, Window::handle collapse, Lua TLS split.
- **Host is `pub(crate)`, not `pub`** — `pub trait Host` would emit `more_private_than_public_item` warnings because `Confirms` / `EngineBridge` / `Cells` / `Timers` are `pub(crate)` types.
- **Skip `tools()` until P2.a.10 lands** — adding a placeholder `tools()` would force a `ToolRuntime` stub in Core that no caller exercises.
- **First-consumer migration covers every method** — landing the trait without consumers triggers `dead_code`.
- **Frontend impls delegate, not re-aggregate** — `impl Host for TuiApp` / `for HeadlessApp` is mechanical: every method is `self.core.X()`.
- **`UiHost` is `pub`, not `pub(crate)`** — the trait lives in the `ui` crate and its return types are all `pub`.
- **`set_focus` over architecture-sketch `focus(&mut self, win)`** — matching the split keeps the trait method names aligned with `Ui`'s inherent surface.
- **`Ui` impls `UiHost` for tests + direct callers** — parallel to `Core` impl `Host`.
- **No `Host` supertrait** — `ui` cannot reference tui-defined `Host`. The trait split is by *crate boundary*, not by feature surface.
- **Skip `fire_win_event` / `overlay_close` / `ui()` migration this commit** — every production `fire_win_event` callsite holds a borrow on `self.core.lua` that conflicts with the `&mut self`-shaped trait dispatch.
- **`Status` collapses `DispatchOutcome` + `MouseAction` into one enum** — `Status::{ Consumed | Capture | Ignored }` is the union.
- **`From<crossterm::event::Event>` over a host-side translator fn** — the conversion is a 1-1 enum re-wrap.
- **`FocusTarget` is a single-variant enum, not a type alias** — future-proof distinction from `HitTarget`.
- **P2.b.4 splits into b.4a / b.4b / b.4c** — three natural single-session units.
- **Kill-ring inspection moves to the caller, not behind a richer Status** — `Window::handle_key`'s old `Option<Option<String>>` shape leaked rendering hacks into the Window primitive's signature.
- **P2.b.5 splits into b.5a / b.5b / b.5c** — full TLS-slot generalization + bulk-migration of ~85 `with_app` callsites is too big for one session.
- **`with_host` / `try_with_host` are `pub(crate)`, `with_ui_host` / `try_with_ui_host` are `pub`** — sibling to the trait visibilities.
- **P2.b.4c splits into c.1…c.6** — outside-in: drop viewport mirrors, click-count, scrollbar drag, wheel scroll, Down/Drag/Up, drag autoscroll.
- **`Window::viewport` is canonical, not the TuiApp shadows** — render-loop already writes `win.viewport` on the same WinId.
- **Scrollbar capture-aware routing in `Ui::dispatch_event` keeps host-side mirror state via post-dispatch read-back** — the cleanest split for c.3.
- **`Ui::hit_test` returns `Scrollbar` for splits leaves now** — doc-comment's "never returns Scrollbar yet" carve-out lifts.
- **The `begin_scrollbar_drag_if_hit` two-call dance retires** — discriminator becomes `Ui::hit_test(row, col, None) == HitTarget::Scrollbar { owner }`.
- **Wheel coalescer tracks the column too** — `Ui::hit_test(row, col, None)` needs the column for overlay/chrome resolution.
- **Routing the wheel through `Ui::hit_test` over `LayoutState::hit_test`** — `Ui::hit_test` is the canonical splits + overlays + scrollbar query.
- **c.5 splits into c.5a / c.5b / c.5c** — Window-capture, host accessors on `UiHost`, unified `Window::handle`.
- **`Ui::resolve_split_mouse` over a `mouse_invoke` closure callback** — returning `Option<(WinId, u8)>` from a separate method avoids closure borrow gymnastics.
- **Window capture replaces `mouse_drag_active`-as-a-flag for routing** — capture is the canonical "is the user mid-gesture" signal.
- **`UiHost` accessors return owned data, not borrowed slices** — owning the return values keeps the trait stateless; mouse events are infrequent enough.
- **`Ui::capture()` over a separate `mouse_drag_active` flag** — capture is set on Down and cleared on Up; semantic span difference is hairline.
- **`drag_autoscroll_since` lives on `Ui`, not `Window`** — autoscroll engagement is a property of the gesture, not the captured pane.
- **Edge detection reads `Ui::win(captured).cursor_line` directly** — lag is at most one tick.
- **TuiApp overrides only `TRANSCRIPT_WIN` end-to-end this commit** — c.5b lands the trait and `TuiApp`'s impl but only `handle_content_mouse` migrates fully.
- **`viewport_for(&self, win)` is `&self`, not `&mut self`** — pure read of `self.win(win)?.viewport`.
- **Option (a) over (b) for c.5c host-arg shape** — drop the host arg from `Window::handle`; ctx carries per-pane data; ARCHITECTURE.md revises.
- **Synthesised `WindowViewport` for the key path** — `EventCtx` carries a full `WindowViewport` but key dispatch reads only `viewport.rect.height`.
- **`Window::handle_key` / `handle_mouse` go `pub(crate)`, not deleted** — every production caller now goes through `Window::handle` but per-event helpers stay reachable for tests.
- **`Buffer::attach` API design lands with the first transcript-pipeline consumer, not as an up-front design pass** — parser-kind enum, `on_block` payload shape, and extmark invariants emerge from migration.
- **L2 harness deferred indefinitely** — `cargo nextest` + `cargo clippy -D warnings` + FEATURES.md walk have caught every drift so far.
- **P2.b.5b trigger is P5.b** — TLS-slot generalization needs a real headless Lua consumer to validate the error path end to end.
- **P2.d.1 lands as relocation, not method on EngineBridge** — `&mut self.core.engine` is the receiver and the body re-borrows `app.core.engine` — free functions avoid the aliasing.
- **Six private agent.rs helpers bump to `pub(super)`, no migration** — cheaper than relocating six more functions.

## Deferrals

- **a.10 (ToolRuntime carve-out)** — split off and waiting on P5.a's `engine::ToolDispatcher` trait shape.
- **P2.b.5b** — TLS-slot generalization to carry either `TuiApp` or `HeadlessApp`, with UiHost-from-headless raising a runtime error. Deferred until P5.b surfaces a headless Lua callback driver.
- **P2.b.5c** — bulk-migrate the remaining ~85 `with_app` callsites to `with_host` / `with_ui_host` and drop the alias. Lands once b.5b is in place.
