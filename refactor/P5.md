# P5 — Tools to Lua

**Status:** in-progress **Started:** 2026-05-01 **Landed:** —

P5 moves the 15 Rust tool implementations in `engine/tools/` into
`runtime/lua/smelt/tools/`. Engine becomes schema + dispatcher only.
Mode gating becomes a Lua-tool `hooks` concern. The `engine/permissions/`
module pulls out to `tui::permissions`; engine ends policy-free.

Sub-phases:

- **P5.a** — Tool dispatcher trait shape. Engine takes
  `Box<dyn ToolDispatcher>` at startup. `ToolSchema` slims; the
  `Tool` trait drops `needs_confirm` / `preflight` /
  `approval_patterns` / `modes` / `decide_override`.
- **P5.b** — Migrate core tools to Lua. 15 tools relocate to
  `runtime/lua/smelt/tools/`; intricate logic stays in Rust as
  `tui::*` capabilities the Lua tools FFI into.
- **P5.c** — Engine cleanup. `engine/permissions/` → `tui::permissions`.
  Drop the multi-agent concept (`Role::Agent`, `EngineEvent::AgentMessage`,
  etc.). `EngineHandle` becomes channels-only. Drop
  `EngineConfig.interactive`.
- **P5.d** — Drop `config.yaml`; all config in `init.lua`. Drop
  `serde_yml` dep + the YAML loader.
- **P5.e** — Protocol rename pass. `Mode → AgentMode`,
  `ExecutePluginTool → ToolDispatch`, etc. Drop the `Plugin` prefix
  from tool defs (one registry now).

## Decisions made

- **Drop dead `Tool` trait methods** (`9c356d8`) — `interactive_only` /
  `modes` / `decide_override` retire; `definitions(interactive: bool)`
  arg drops.
- **Lift `Tool::is_mcp` to a `ToolEntry` data flag** (`030aecd`) — set
  at registration via `register` / `register_mcp`.
- **Collapse `needs_confirm` / `approval_patterns` / `preflight` into
  `Tool::evaluate_hooks(&args) -> protocol::PluginToolHooks`**
  (`a26a42e`) — three trait methods fold into one returning the
  shape plugin tools already use.
- **Extract `Turn::resolve_decision` shared by core+plugin paths**
  (`c27dca1`) — the `permissions.decide → runtime-approvals upgrade`
  block was duplicated; folded into one associated fn.
- **Introduce `ToolDispatcher` trait + route engine through it**
  (`db2ed43`) — engine never touches `&dyn Tool` directly; every
  per-call decision goes through `definitions / contains / is_mcp /
  evaluate_hooks / dispatch`.
- **Drop `&Permissions, Mode` from `ToolDispatcher::definitions`**
  (`48e742d`) — dispatcher returns the unfiltered list; engine
  applies mode-permission filtering externally via `is_mcp`.
- **P5.e protocol rename pass** (`2320338`) — `Mode → AgentMode`,
  `SetMode → SetAgentMode`, `ExecutePluginTool → ToolDispatch`,
  `EvaluatePluginToolHooks → ToolHooksRequest`,
  `PluginTool{Result,HooksResult} → Tool{Result,HooksResponse}`,
  `Plugin` prefix drops from tool def types (one registry now).

### P5.b Lua tool migrations

- **`glob` → Lua** (`9be6f95`) — composes new `tui::fs::glob`
  capability via `smelt.fs.glob`. `tools/` directory introduced.
- **`grep` → Lua** (`72fc354`) — composes `smelt.grep.run` (ripgrep)
  with `smelt.process.run("grep", ...)` fallback.
- **`load_skill` → Lua** (`5f5e841`) — composes new `smelt.skills.content`
  over a shared `Arc<SkillLoader>` clone parked on `Core.skills`.
- **`list_agents` → Lua** (`48536df`) — composes new
  `smelt.agent.{discover,my_pid}` over `engine::registry`. Gated on
  `smelt.engine.multi_agent()`.
- **`stop_agent` → Lua** (`53909e3`) — composes new
  `smelt.agent.{find_by_id,is_in_tree,kill,my_pid}`. Owner-only check
  preserved.
- **`ask_user_question` relocated `plugins/` → `tools/`** (`2954db1`) —
  embedded module path renamed to `smelt.tools.ask_user_question`.
- **`peek_agent` + `message_agent` → Lua** (`1d977a2`) — composes new
  `smelt.agent.{send_query,send_message,my_id,my_slug}` over
  `engine::socket::send_{query,message}_blocking` blocking wrappers.
  Drops `MultiAgentToolConfig.{agent_id,slug}` and
  `MultiAgentConfig.agent_id`.
- **`web_search` → Lua + `tui::http` POST/cache/UA** (`17ba19e`) —
  composes new `smelt.http.{post,cache.{get,put},random_user_agent}`
  + `smelt.html.parse_ddg_results`. `tui::http::cache` relocates from
  `engine/tools/web_cache.rs` (both implementations read/write the
  same `<cache_dir>/web/<hash>` files until `web_fetch` retires).
- **`FileStateCache` plumbed through `engine::start` + parked on Core**
  (`711492d`) — lifts `FileStateCache` / `FileState` /
  `file_mtime_ms` / `staleness_error` from `pub(crate)` to `pub`;
  `engine::start(config, files)` takes the cache as a param so engine
  + Lua see the same observations. `smelt.fs.file_state.*` exposes
  the cache to Lua.
- **`write_file` → Lua** (`25e31eb`) — composes
  `smelt.fs.{exists,mkdir_all,write,try_flock,file_state.*}` +
  `smelt.notebook.is_notebook_path` + new `smelt.path.display`. Lifts
  `try_flock` / `FlockGuard` / `display_path` to `pub`. Lua-side
  flock guard is a userdata wrapper with `:release()` + `__gc`.
- **`edit_file` → Lua** (`da42a63`) — composes
  `smelt.fs.{read,write,try_flock,file_state.*}`; pure-Lua
  `replace_first` / `replace_all` over `string.find(..., 1, true)`.
- **`read_file` → Lua** (`47cc6d3`) — composes new
  `smelt.image.{is_image_file,read_as_data_url}` (new `lua/api/image.rs`)
  + new `smelt.notebook.read` (over public
  `engine::tools::notebook::render_notebook_text` wrapper) +
  `smelt.fs.{read,file_state.*}`. Pure-Lua `dedup_stub` /
  `format_text_window` reproduce the engine semantics. Drops dead
  `tools::int_arg` and `notebook::is_notebook` helpers.
- **`edit_notebook` → Lua** (`991b070`) —
  `runtime/lua/smelt/tools/notebook_edit.lua` thin wrapper over a new
  `smelt.notebook.apply_edit(args)` FFI (over a new `pub`
  `engine::tools::notebook::apply_edit` returning
  `NotebookEditOutcome { message, metadata }`). Intricate JSON cell
  munging stays Rust; Lua owns schema, hook, flock, metadata
  passthrough via a new `super::json_to_lua_value` helper. Drops
  `ToolContext.file_locks` + `engine::tools::FileLocks` (last
  consumer gone — flock alone serializes concurrent edits).
- **`exit_plan_mode` extracted to `tools/exit_plan_mode.lua`** (this session) —
  the Plan-mode tool body (schema, plan-name generator, `save_plan`,
  confirm dialog) relocates verbatim from `plugins/plan_mode.lua` into
  a `register / unregister` module the plugin requires. Plan-mode hook
  driver shrinks to ~60 LOC.
- **`web_fetch` → Lua + `tui::html::to_markdown`** (`c1c9de9`) —
  `runtime/lua/smelt/tools/web_fetch.lua` composes
  `smelt.http.{get,cache.{get,put},random_user_agent}` +
  `smelt.html.{title,links,to_text,to_markdown}` +
  `smelt.image.data_url_from_bytes` + `smelt.engine.ask` (with
  `smelt.task.alloc/wait` parking the coroutine until
  `EngineAskResponse` fires). The full HTML→markdown converter
  (`ParsedHtml::to_markdown` + helpers) relocates from
  `engine/tools/web_shared.rs` into `tui::html` as
  `Markdown { title, content, links }`. The `smelt.http.get` body
  switches from lossy UTF-8 to a byte-safe Lua string so binary
  responses (image content-types) survive the boundary unchanged.
  LLM extraction routes through `AuxiliaryTask::Btw` — falls back
  to the primary model when no btw aux is configured (was the
  primary unconditionally before). `engine/tools/{web_fetch,
  web_shared}.rs` retire; `web_cache.rs` keeps its TTL setter for
  pricing's metadata cache. `ToolContext` drops the
  `provider / model / api` fields (web_fetch was their only
  consumer); `Provider::extract_web_content` retires.
