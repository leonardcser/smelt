# P1 — UI primitives (the load-bearing phase)

**Status:** in-progress
**Started:** 2026-04-29
**Landed:** (in progress — P1.0, P1.a foundation, and P1.b complete; P1.c..P1.g remain)

P1 is the long phase. It rebuilds the `ui` crate around the target
primitives (`Buffer`, `LayoutTree`, `Overlay`, `Window`, `Theme`,
`UiHost`, `Ui` facade) and absorbs the structural deletions that P0
deferred. Sub-phases land independently, each with the tree green at
the boundary; this log accumulates entries as they close.

## What shipped vs what was planned

### P1.0 — Theme registry + paired structural deletions ✅

Landed: `decb0ab` → `e489a79` (12 commits + status updates).

Planned: nvim-style highlight-group registry (`ui::Theme { groups,
links }`) replacing the host-side `crate::theme::*` atomics; pair
the deletion of `crates/tui/src/theme.rs` constants with the
registry's introduction.

Shipped exactly that. `ui::Theme` lives on `Ui`, threads through
`DrawContext`, and is populated each frame from `theme.is_light()`
+ `theme.accent()` + `theme.slug()`. Renderers read `ctx.theme.get(group)`
instead of host constants. The full atomic set on `tui::theme` (BG,
ACCENT, SLUG) collapsed onto `ui::Theme` itself; Lua-driven mutations
flow through `with_app(|app| app.ui.theme())`.

`crate::theme::*` after P1.0:
- `populate_ui_theme(&mut Theme)` — initializes `Smelt*` groups from
  `is_light` / `accent` / `slug`.
- `detect_background(&mut Theme)` — OSC 11 / `$COLORFGBG` probe.
- `PRESETS` + `preset_by_name` — accent presets.
- `DEFAULT_ACCENT` re-export from `ui::theme::DEFAULT_ACCENT`.

42 call-site migrations, `Snapshot` snapshot type eliminated,
`ColorRole` expanded with Agent/Success/ErrorMsg/Apply/Plan/Exec/
Heading/ReasonLow/Med/High/Max. Syntect's light/dark hint moved to
`content/highlight/mod.rs` as a self-contained mirror updated by
`populate_ui_theme()` (avoids threading `&Theme` through 14 syntax
call sites for one branch).

### P1.a — Buffer foundation (laid; structural completion deferred)

Landed: `50b8923`, `67603de`, `26701d3`, `385e9d0`. Foundation
shipped: extmark + namespace model, YankSubst, soft-wrap cache,
`BufferParser` rename + `on_attach` hook.

Planned (per REFACTOR.md § P1.a): replace `lines`/`spans`/`marks`
with extmark model; add `Buffer::attach(spec)` parser hooks;
delete `transcript_cache.rs` + `BufferFormatter` trait; merge
`edit_buffer.rs` into `Buffer`.

Shipped:
- `ui::Buffer` storage is now `BTreeMap<ExtmarkId, Extmark>` per
  `Namespace`. `add_highlight` / `set_decoration` / `set_virtual_text`
  / `set_mark` are thin wrappers over `set_extmark` in well-known
  namespaces (`NS_HIGHLIGHTS`, `NS_DECORATIONS`, `NS_VIRT_TEXT`,
  `NS_MARKS`).
- `YankSubst::{ Empty | Static(String) }` extmark field +
  `Buffer::yank_text_for_range` helper. No callers yet (intended
  consumers — hidden-thinking elision, attachment expansion — turned
  out to be wrong fits; see Decisions).
- `Buffer::wrap_at(width)` soft-wrap cache keyed by `(changedtick,
  width)`. No callers yet (rendering is pre-wrapped via DisplayLine).
- `BufferFormatter` trait → `BufferParser`; `render` method →
  `parse`; `with_formatter` builder → `attach`; `set_formatter` →
  `set_parser`. `BufferParser::on_attach(&mut Buffer)` lifecycle
  hook (default no-op) for parsers to register custom namespaces.

Deferred to P1.a-tail (gated on transcript-pipeline migration onto
`BufferParser`, itself a multi-session keystone):
- `Buffer::attach(spec)` parser-hook system replacing `BufferFormatter`
  trait + `transcript_cache.rs` IR cache.
- Transcript renderers (`transcript_present/*.rs`) onto `BufferParser`
  — each `Block` kind becomes its own parser, `BlockArtifact` becomes
  per-block Buffer, `TranscriptSnapshot` composes from per-block
  Buffers.
- `transcript_cache.rs` deletion (per-parser IR caches replace it).
- `edit_buffer.rs` merge into `Buffer` (~250 references; pairs
  naturally with P1.d when vim state machine decomposes).

### P1.b — LayoutTree ✅

Landed: B.1 → B.4 (`70ad2e0`, `5e1d81c`, `14612d8`, `e615c12`,
`2a157da`).

Planned (per REFACTOR.md § P1.b): `Vbox | Hbox | Leaf(WinId)` with
chrome on containers; `Item = (Constraint, LayoutTree)`; constraints
`Length / Percentage / Ratio / Min / Max / Fill / Fit`.

Shipped exactly that:
- B.1 — `Constraint` expanded from `{Fixed, Pct, Fill}` to the seven
  target variants. Three-pass resolution (hard sizes → ratios →
  fill). `Fit` stubbed to `Fill` until leaves expose natural size.
  Renamed `Fixed → Length`, `Pct → Percentage`. 8 consumer files
  updated.
- B.2 — `gap`/`border`/`title` chrome on `Split`; `vbox`/`hbox`/`leaf`
  constructors and `with_gap`/`with_border`/`with_title` builders.
  All in-file tests rewritten on the constructors. Dummy "gap"
  leaf in `content/layout.rs` replaced with `.with_gap(1)`.
- B.3 — `Item = (Constraint, LayoutTree)`; constraint hoisted out
  of `Leaf` onto parent `items`. `Leaf(WinId)` (string names gone).
  `resolve_layout` returns `HashMap<WinId, Rect>`. Production
  consumers use `TRANSCRIPT_WIN`/`PROMPT_WIN` constants.
- B.4 — `Split` replaced with explicit `Vbox`/`Hbox` variants
  sharing a `Chrome` struct. `Direction` enum deleted.

Deviation from spec: `SeparatorStyle` (planned as a `Chrome`
field) deferred — needs new enum + render-time wiring; will land
in P1.c alongside Overlay chrome.

### P1.c — Overlay replacing Float (in progress, scoping)

Per REFACTOR.md § P1.c: `Overlay { layout: LayoutTree, anchor:
Anchor, z: u16, modal: bool }` with rich `Anchor` enum (`Screen` /
`Cursor` / `Win { target, attach }`). Deletes `Float`/`FloatId`,
`Placement`, `PanelWidget` trait + `dialog.rs` panel multiplexing.

Not yet started in code. Scoping in STATUS.

## Decisions made while coding

- **Theme registry collapses onto a single owned struct, no global
  atomics.** Original P1.0 plan kept the `tui::theme` atomics as the
  source of truth and mirrored them into `ui::Theme` per-frame.
  Mid-phase, we collapsed: `ui::Theme` itself owns `is_light`,
  `accent`, `slug` as fields; per-frame `populate_ui_theme()` reads
  those and rewrites the `Smelt*` groups. Lua API closures access
  theme via `with_app(|app| app.ui.theme())`. **Why:** removed a
  global mutable state path entirely; now `Ui` is the single owner.
- **Syntect light/dark hint stays as a self-contained atomic in
  `content/highlight/mod.rs`.** Threading `&Theme` through 14 syntect
  call sites for a single boolean was disproportionate. Kept a
  module-local atomic that `populate_ui_theme()` updates each frame.
- **P1.a closes as "foundation laid"; structural completion
  deferred.** The deeper P1.a deliverables (parser-hook `attach(spec)`,
  `transcript_cache.rs` deletion, `edit_buffer.rs` merge) are gated
  on the transcript-pipeline migration onto `BufferParser`, which
  is itself a multi-session keystone. Rather than block all of P1
  on it, closed P1.a as foundation and moved to P1.b (independent).
  See STATUS § "Deferred to P1.a-tail."
- **`YankSubst` consumers turned out to be wrong fits.** The two
  intended consumers (hidden-thinking elision, prompt attachment
  expansion) don't actually fit: hidden-thinking copy goes through
  `TranscriptSnapshot` which bypasses Buffer entirely; attachment
  expansion happens at submit, not on copy. Closed both as "wrong
  fit" in STATUS. The API still landed because the eventual
  consumers (transcript blocks once migrated to BufferParser) will
  use it.
- **`Buffer::wrap_at` has no caller yet.** Same situation —
  rendering pre-wraps via `DisplayLine` before reaching the buffer.
  Wrap state today still lives in `WindowView`. The cache is
  correct API shape; consumers come with the transcript migration.
- **B.2 used builder constructors instead of struct-literal
  field-defaults.** `Direction` has no sensible Default and existing
  code constructed `Split { direction, children }` literally.
  Adding chrome fields with `..Default::default()` would have
  required deriving `Default` on `Split` (awkward). Instead added
  `LayoutTree::vbox(items)` / `hbox(items)` / `leaf(win)` builders
  + `with_gap` / `with_border` / `with_title`. **Side effect:** the
  builders are exactly the eventual `Vbox`/`Hbox` variant
  constructors needed in B.4 — saved a second migration.
- **Replaced "gap" leaf with `.with_gap(1)` chrome.** `content/layout.rs`
  modeled inter-pane spacing as a third leaf named "gap" of `Length(1)`
  that nothing read. With chrome support landed in B.2, switched to
  `.with_gap(1)`. Cleans up a workaround and removes a dead leaf
  before B.3's WinId migration (one less non-window leaf to handle).
- **Test markers (`A`/`B`/`C` constants) instead of synthetic
  `WinId::new`.** Layout tests in B.3+ need `WinId` values that don't
  collide with `PROMPT_WIN(0)` / `TRANSCRIPT_WIN(1)`. Used module-level
  `const A: WinId = WinId(100)`-style markers. Cleaner read in tests
  than `WinId(2)` + comment, and the offset signals "synthetic test
  id, not a real reserved one."

## Files / types / functions changed

P1.0:
- Added: `crates/ui/src/theme.rs` (`Theme` registry type, accent/slug/light
  fields, presets); `ColorRole::{ Agent, Success, ErrorMsg, Apply, Plan,
  Exec, Heading, ReasonLow, ReasonMed, ReasonHigh, ReasonMax }`.
- Deleted: `crate::theme::Snapshot` snapshot type; `tui::theme` atomic
  globals (`BG_LIGHT`, `ACCENT`, `SLUG`, accessors).
- Restructured: 42 renderer call sites read `ctx.theme.get(group)`
  instead of host constants; `Ui` owns `theme: Theme` and threads it
  through `DrawContext`.

P1.a:
- Added: `ui::buffer::{ NsId, ExtmarkId, Extmark, ExtmarkPayload,
  ExtmarkOpts, YankSubst, WrapResult }`; `Buffer::{ create_namespace,
  set_extmark, del_extmark, clear_namespace, extmarks, yank_text_for_range,
  wrap_at }`; `BufferParser` trait + `on_attach` hook; well-known
  namespace constants `NS_HIGHLIGHTS` / `NS_DECORATIONS` / `NS_VIRT_TEXT` /
  `NS_MARKS`.
- Renamed: `BufferFormatter` → `BufferParser`; `render` → `parse`;
  `with_formatter` → `attach`; `set_formatter` → `set_parser`;
  `has_formatter` → `has_parser`. Tui-side `ModeFormatter` →
  `ModeParser`; `into_formatter` → `into_parser`.
- Restructured: `Buffer` storage moved to per-namespace
  `BTreeMap<ExtmarkId, Extmark>`; per-line getters
  (`highlights_at` / `decoration_at` / `virtual_text_at` / `get_mark`)
  read directly from extmarks; `highlights_arc` / `decorations_arc`
  materialize on demand with a `(changedtick + marks_tick)` cache.
  `BufferView::sync_from_buffer` and `dialog::build_panels` now
  take `&mut Buffer` / `&mut dyn BufferResolver`.

P1.b:
- Added: `Constraint::{ Min, Max, Ratio, Fit }`; `LayoutTree::{ Vbox,
  Hbox }` variants; `Chrome { gap, border, title }` struct; `Item =
  (Constraint, LayoutTree)` type alias; `LayoutTree::{ vbox, hbox,
  leaf, with_gap, with_border, with_title }` constructors / builders.
- Renamed: `Constraint::Fixed` → `Length`; `Constraint::Pct` →
  `Percentage`.
- Deleted: `LayoutTree::Split { direction, children }` (replaced by
  `Vbox`/`Hbox`); `Direction` enum; the dummy `gap` leaf in
  `content/layout.rs`.
- Restructured: `Leaf { name: String, constraint: Constraint }` →
  `Leaf(WinId)`; constraint hoisted onto parent `items`;
  `resolve_layout` returns `HashMap<WinId, Rect>` instead of
  `HashMap<String, Rect>`. `Ui::resolve_splits` updated to match.
