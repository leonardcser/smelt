# P3 — Rust capabilities + Lua API split

**Status:** in-progress **Started:** 2026-04-30 **Landed:** —

P3 puts the generic Rust capabilities behind named modules under
`tui::*` (parse / process / subprocess / fs / http / html / notebook /
grep / path / fuzzy / permissions) and splits the Lua bindings to one
file per `smelt.*` namespace under `crates/tui/src/lua/api/<name>.rs`.
Mostly mechanical; unblocks P4 (Lua takes the UX) and P5 (tools to
Lua). Sub-phases land independently.

## Sub-phases landed

- **P3.a (`tui::fuzzy`)** (`b537d1a`) — `fuzzy.rs` becomes a folder module; `completer/score.rs` folds in as `tui::fuzzy::score`; `completer::{mod,history}` consume via `crate::fuzzy::score::*`.
- **P3.b (`smelt.timer` + `smelt.defer`)** (`63b4d12`) — register fn moves to `crates/tui/src/lua/api/timer.rs`; dispatch.rs umbrella delegates via `super::timer::register`.
- **P3.b (`smelt.cell` + `smelt.au`)** (`a106b79`) — cell registry bindings + `CellHandle` userdata move to `lua/api/cell.rs`; `au.{on,fire}` thin alias moves to `lua/api/au.rs`; dispatch.rs trims to the remaining six namespaces (cmd / keymap / task / tools / statusline / spawn).

## Decisions made

- **`tui::fuzzy` is a folder module, not a flat file** (`b537d1a`) — `mod.rs` keeps `fuzzy_score`; `score.rs` carries the completer ranking helpers (`query_match_score` / `split_words` / `recency_bonus`). They share the same ranking surface and the completer is the only consumer of `score::*` today; co-locating keeps the future `smelt.fuzzy` Lua binding (P3.c) one import away.
