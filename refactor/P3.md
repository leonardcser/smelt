# P3 — Rust capabilities + Lua API split

**Status:** in-progress **Started:** 2026-04-30 **Landed:** —

P3 puts the generic Rust capabilities behind named modules under
`tui::*` (parse / process / subprocess / fs / http / html / notebook /
grep / path / fuzzy / permissions) and splits the Lua bindings to one
file per `smelt.*` namespace under `crates/tui/src/lua/api/<name>.rs`.
Mostly mechanical; unblocks P4 (Lua takes the UX) and P5 (tools to
Lua). Sub-phases land independently.

## Sub-phases landed

- **P3.a (`tui::fuzzy`)** (`b537d1a`) — `fuzzy.rs` becomes a folder module; `completer/score.rs` folds in as `tui::fuzzy::score`; `completer::{mod,history}` consume via `crate::fuzzy::score::*`.
- **P3.b (`smelt.timer` + `smelt.defer`)** (`63b4d12`) — register fn moves to `crates/tui/src/lua/api/timer.rs`; dispatch.rs umbrella delegates via `super::timer::register`.
- **P3.b (`smelt.cell` + `smelt.au`)** (`a106b79`) — cell registry bindings + `CellHandle` userdata move to `lua/api/cell.rs`; `au.{on,fire}` thin alias moves to `lua/api/au.rs`; dispatch.rs trims to the remaining six namespaces (cmd / keymap / task / tools / statusline / spawn).
- **P3.b (`smelt.cmd` + `smelt.keymap`)** (`5c81524`) — `cmd.{register,run,list}` and `keymap.{set,help}` move to `lua/api/{cmd,keymap}.rs`; dispatch.rs umbrella shrinks to task / tools / statusline / spawn.
- **P3.b (`smelt.task` + `smelt.tools` + `smelt.statusline` + `smelt.spawn`)** (`e36fc68`) — final four dispatch namespaces split into `lua/api/{task,tools,statusline,spawn}.rs`; `dispatch.rs` umbrella retires; `lua/api/mod.rs` orchestrates the per-namespace registers directly.
- **P3.b (`smelt.theme` + `smelt.buf` + `smelt.win` + `smelt.ui` + `smelt.prompt` + `smelt.settings` + `smelt.vim`)** (`f129f07`) — `widgets.rs` umbrella splits into `lua/api/{theme,buf,win,ui,prompt,settings,vim}.rs`; `mod.rs` ui::Theme references disambiguate to `::ui::Theme` to dodge the new local `mod ui` shadowing the external crate.
- **P3.b (`smelt.transcript` + `smelt.engine` + `smelt.session` + `smelt.process` + `smelt.shell` + `smelt.agent` + `smelt.permissions` + `smelt.fuzzy` + `smelt.history`)** (`f53d5f9`) — `state.rs` umbrella splits into `lua/api/{transcript,engine,session,process,shell,agent,permissions,fuzzy,history}.rs`; `lua/api/` is now one file per top-level `smelt.<name>` namespace, no umbrellas left.
- **P3.b (`smelt.diff` + `smelt.syntax` + `smelt.bash` + `smelt.notebook`)** (`9d40423`) — `lua/render_ops.rs` umbrella splits into `lua/api/{diff,syntax,bash,notebook}.rs`; `lua/mod.rs` drops the `render_ops` module; `register_api` invokes the four register fns directly. Resolves the open INVENTORY question — diff / syntax / bash / notebook stay as separate top-level `smelt.<name>` bindings (per-language as-is) rather than folding under a single `smelt.parse`.
- **P3.b (`smelt.clipboard`)** (`cc9d279`) — inline write-only binding in `register_api` carves out into `lua/api/clipboard.rs`; adds `smelt.clipboard.read()` (was missing per FEATURES); legacy callable form (`smelt.clipboard("text")` ↦ write) preserved via a `__call` metatable.
- **P3.b (`smelt.confirm`)** (`050c8b8`) — `lua/confirm_ops.rs` becomes `lua/api/confirm.rs`; `lua/mod.rs` drops the `confirm_ops` module; `register_api` invokes `confirm::register` like the other per-namespace files. Last umbrella-adjacent file in `lua/` retires.
- **P3.c (`smelt.mode`)** (`ad9eccc`) — `lua/api/mode.rs` exposes `get / set / cycle` over `protocol::Mode`; `smelt.engine.{mode,set_mode}` retire (Decision #9: `smelt.mode` lives at top-level so it does not collide with future `smelt.subprocess`); `runtime/lua/smelt/plugins/plan_mode.lua` callers updated.

## Decisions made

- **`tui::fuzzy` is a folder module, not a flat file** (`b537d1a`) — `mod.rs` keeps `fuzzy_score`; `score.rs` carries the completer ranking helpers (`query_match_score` / `split_words` / `recency_bonus`). They share the same ranking surface and the completer is the only consumer of `score::*` today; co-locating keeps the future `smelt.fuzzy` Lua binding (P3.c) one import away.
