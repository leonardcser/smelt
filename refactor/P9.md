## Decisions made

- **Well-known window IDs moved from `ui` to `app` (`2026-05-04`).**
  `PROMPT_WIN`, `TRANSCRIPT_WIN`, and `PROMPT_EDIT_BUF` live in
  `app/well_known.rs`. `ui` no longer knows that `WinId(0)` means prompt.
  `win_open_split` is now collision-tolerant (`while wins.contains_key(...)`)
  and `next_win_id` starts at `0`, so reserved-id windows can be created
  before or after regular windows without collision.

- **Prompt data file renamed to `prompt_buf.rs` in P9.a.** Matches
  `transcript_buf.rs`; both project into `Buffer`. Done early so P9.c
  rewrites the file under its final name.

- **Pre-existing clippy warnings fixed as part of P9.a lint gate.**
  `len_without_is_empty` on `FileStateCache` and `Timers`;
  `new_without_default` on `BashHighlighter`, `StreamParser`,
  `Transcript`, `LuaRuntime`; derivable `Default` on `ThrobberItem`;
  `arc_with_non_send_sync` allows on `tui/src/lua/mod.rs`. These were
  latent since P8 but blocked `-D warnings`.
