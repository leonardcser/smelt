# P9 — Final cleanup

**Status:** in-progress **Started:** 2026-05-04 **Landed:** P9.a 2026-05-04

## Decisions made

- **Well-known window IDs moved from `ui` to `app` (`2026-05-04`).**
  `PROMPT_WIN`, `TRANSCRIPT_WIN`, and `PROMPT_EDIT_BUF` live in
  `app/well_known.rs`. `ui` no longer knows that `WinId(0)` means prompt.
  `win_open_split` is now collision-tolerant (`while wins.contains_key(...)`)
  and `next_win_id` starts at `0`, so reserved-id windows can be created
  before or after regular windows without collision.

- **Prompt data file renamed to `prompt_buf.rs` in P9.a.** Matches
  `transcript_buf.rs`; both project into `Buffer`. Done early so P9.c
  rewrites the file under its final name.

- **Pre-existing clippy warnings fixed as part of P9.a lint gate.**
  `len_without_is_empty` on `FileStateCache` and `Timers`;
  `new_without_default` on `BashHighlighter`, `StreamParser`,
  `Transcript`, `LuaRuntime`; derivable `Default` on `ThrobberItem`;
  `arc_with_non_send_sync` allows on `tui/src/lua/mod.rs`. These were
  latent since P8 but blocked `-D warnings`.

- **`buffer::SpanStyle` unified with `grid::Style` (`2026-05-04`).**
  `ui/buffer.rs` now defines `pub type SpanStyle = super::grid::Style;`
  instead of a separate 5-field struct. This fixes the latent strikethrough
  rendering bug (crossed-out text was silently dropped at three conversion
  sites) and removes ~50 LOC of mechanical conversion. `merge_span_style`
  now OR-merges `underline` and `crossedout` from highlight extmarks.

- **Persisted transcript caches deleted (`2026-05-04`).**
  `transcript_cache.rs` (which held `PersistedLayoutCache`, `RenderCache`, and
  `ToolOutputRenderCache`) is gone. The persisted layout and render caches
  are no longer saved to or loaded from disk — `Buffer::ensure_rendered_at`
  will handle caching once the transcript pipeline migration completes.
  `BlockArtifact` and `DisplayBlock` remain in `core` for now; they will
  move or be deleted when the renderer-to-Buffer migration lands.

## Decisions made (continued)

- **Tool output rendering moved to Lua `render` hook with `RenderCtx`**
  (`2026-05-04`). Instead of `build_cache` returning opaque JSON mode specs
  that Rust dispatches through `ModeSpecRenderer`, each tool now registers a
  `render` function that receives a `RenderCtx` userdata:
  ```lua
  render = function(args, output, width, ctx)
    ctx:text(output.content, output.is_error)
  end
  ```
  `RenderCtx` exposes Rust rendering primitives directly:
  `ctx:text()`, `ctx:diff()`, `ctx:file()`, `ctx:markdown()`,
  `ctx:code()`, `ctx:newline()`. This gives Lua full compositional
  flexibility; Rust caches expensive work (diff LCS, syntax tokenization)
  internally without exposing cache keys to Lua.

  Deleted: `build_cache` hook, `ToolOutput.render_cache`,
  `ModeSpecRenderer`, `lua_value_to_json` (public), `tool_previews.rs`.
