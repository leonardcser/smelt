# P6 ‚Äî Streaming + lifecycle polish

**Status:** in-progress **Started:** 2026-05-02 **Depends on:** P5 completed

Lock down the engine‚Üíbuffer streaming path and the lifecycle gates
(confirms gate, cooperative cancel, dialog stacking).

Sub-phases:

- **P6.a** ‚Äî Streaming pipeline hardening. ‚úÖ Completed.
  Verify `EngineEvent::TextDelta` ‚Üí `EngineBridge` ‚Üí `Buffer::append`
  never drops or reorders chunks. Added 18 `StreamParser` unit tests
  covering chunk boundary independence, structural boundary detection
  (code blocks, tables, paragraph breaks), thinking‚Üítext transition,
  tool/exec lifecycle, and the no-drop invariant (char-by-char feed).
  Buffer's `on_block` callback is still gated on P1.a-tail (`Buffer::attach`).
- **P6.b** ‚Äî Confirms gate. `EngineBridge` checks `Confirms::is_clear()`
  before pulling the next event from `engine.event_rx`. One gate, no
  secondary blocking logic.
- **P6.c** ‚Äî Cooperative cancel. Each Lua coroutine task carries a
  `CancellationToken`. Cancel = token flips, in-flight async Rust calls
  return `Err(Canceled)`, coroutine resumes with that error so normal Lua
  `pcall` flow handles cleanup.
- **P6.d** ‚Äî Dialog stacking + Esc chain. `Ui::overlays` z-stack: open
  order = z order, modal-on-top blocks focus to lower overlays. Esc chain
  = focused Window's `handle_key` first; if `Ignored`, `WinEvent::Dismiss`
  fires on the enclosing Overlay; Lua handles.
- **P6.e** ‚Äî Cursor shape global. Single `Ui::cursor_shape`, not
  per-Window. Updated on focus change.
- **P6.f** ‚Äî Cell-event fan-out tests. Every state-changing event in the
  engine pipeline reaches the right cell setter (`agent_mode`, `model`,
  `reasoning`, `tokens_used`, `turn_complete`, `confirm_requested`,
  `history`, ‚Ä¶). Each cell name has at least one built-in subscriber test
  asserting the callback runs. The `smelt.au.*` alias is exercised the
  same way.

## Decisions made

- **P6.a ‚Äî `parse_marker` trace tests as `StreamParser` unit tests**
  (`<this session>`) ‚Äî rather than adding an intrusive trace vector to
  `StreamParser`, the tests treat `BlockHistory` as the trace surface:
  feed deltas, assert the exact sequence of `Block` variants and
  `Status::Streaming` ‚Üí `Status::Done` transitions. This documents
  current behavior without modifying the hot path. `Block` gained
  `Debug + PartialEq` derives to enable direct equality assertions.
  One known limitation is documented: code fences split *mid-line*
  across chunks are not detected as fences because `StreamParser`
  syncs per-chunk and fences are recognized at line boundaries.
