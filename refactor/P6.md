# P6 — Streaming + lifecycle polish

**Status:** in-progress **Started:** 2026-05-02 **Depends on:** P5 completed

Lock down the engine→buffer streaming path and the lifecycle gates
(confirms gate, cooperative cancel, dialog stacking).

Sub-phases:

- **P6.a** — Streaming pipeline hardening. ✅ Completed.
  Verify `EngineEvent::TextDelta` → `EngineBridge` → `Buffer::append`
  never drops or reorders chunks. Added 18 `StreamParser` unit tests
  covering chunk boundary independence, structural boundary detection
  (code blocks, tables, paragraph breaks), thinking→text transition,
  tool/exec lifecycle, and the no-drop invariant (char-by-char feed).
  Buffer's `on_block` callback is still gated on P1.a-tail (`Buffer::attach`).
- **P6.b** — Confirms gate. ✅ Completed.
  `EngineBridge::recv` returns `pending()` and `try_recv` returns
  `Err(Empty)` while `Confirms::is_clear()` is false. The two outer
  guards in `mod.rs` (`select!` branch + drain-loop `if`) are removed.
  `focused_overlay_blocks_agent()` is dropped from `dispatch_control`
  in `agent.rs` — the single `Confirms` gate is the only engine-drain
  mechanism. `Host::confirms()` retires (all callers reach
  `core.confirms` directly).
- **P6.c** — Cooperative cancel. Each Lua coroutine task carries a
  `CancellationToken`. Cancel = token flips, in-flight async Rust calls
  return `Err(Canceled)`, coroutine resumes with that error so normal Lua
  `pcall` flow handles cleanup.
- **P6.d** — Dialog stacking + Esc chain. `Ui::overlays` z-stack: open
  order = z order, modal-on-top blocks focus to lower overlays. Esc chain
  = focused Window's `handle_key` first; if `Ignored`, `WinEvent::Dismiss`
  fires on the enclosing Overlay; Lua handles.
- **P6.e** — Cursor shape global. Single `Ui::cursor_shape`, not
  per-Window. Updated on focus change.
- **P6.f** — Cell-event fan-out tests. Every state-changing event in the
  engine pipeline reaches the right cell setter (`agent_mode`, `model`,
  `reasoning`, `tokens_used`, `turn_complete`, `confirm_requested`,
  `history`, …). Each cell name has at least one built-in subscriber test
  asserting the callback runs. The `smelt.au.*` alias is exercised the
  same way.

## Decisions made

- **P6.a — `parse_marker` trace tests as `StreamParser` unit tests**
  (`<this session>`) — rather than adding an intrusive trace vector to
  `StreamParser`, the tests treat `BlockHistory` as the trace surface:
  feed deltas, assert the exact sequence of `Block` variants and
  `Status::Streaming` → `Status::Done` transitions. This documents
  current behavior without modifying the hot path. `Block` gained
  `Debug + PartialEq` derives to enable direct equality assertions.
  One known limitation is documented: code fences split *mid-line*
  across chunks are not detected as fences because `StreamParser`
  syncs per-chunk and fences are recognized at line boundaries.
- **P6.b — `Arc<AtomicBool>` shared flag for confirms gate**
  (`<this session>`) — `Confirms` owns an `Arc<AtomicBool>` that
  `register`/`take` update; `EngineBridge` clones it at construction
  and reads it in `recv`/`try_recv`. This avoids a self-referential
  struct in `Core` (both `confirms` and `engine` are fields of the
  same struct). The `Host` trait drops `confirms()` since all
  consumers already access `core.confirms` directly.
