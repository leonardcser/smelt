# P4 — Lua takes the UX

**Status:** done **Started:** 2026-04-30 **Landed:** 2026-05-01

P4 moves every "what does smelt look-behave like" decision out of `tui`
into `runtime/lua/smelt/`. Rust shrinks to capability provision and
pixel pushing.

Sub-phases (interleave-friendly except for P4.b which is gated on
P1.a-tail `Buffer::attach`):

- **P4.a** — `runtime/lua/smelt/` layout. Create `widgets/`, `dialogs/`,
  `colorschemes/` directories and seed files.
- **P4.b** — transcript and diff parsers in Lua. Gated on `Buffer::attach`.
- **P4.c** — reactive statusline.
- **P4.d** — dialogs orchestrated in Lua.
- **P4.e** — slash commands fully Lua.
- **P4.f** — modes registry in Lua.

## Decisions made

- **Slash commands stay one-file-per-command in `plugins/`; no `commands.lua` umbrella** — collapsing 9 self-contained plugins into a single file added bookkeeping without payoff. `cmd.lua` keeps its role as the framework helper. `predict.lua` stays in `plugins/` (no `hooks/` dir for one file). `plan_mode.lua` split shape: tool → P5.b's `tools/exit_plan_mode.lua`; mode hook stays in `plugins/plan_mode.lua`.
- **`prompt_picker.lua` stays separate from `picker.lua`** — both relocate to `widgets/` as siblings. The prompt-docked variant is a non-focusable float that re-filters on every prompt keystroke; the generic picker is a focusable modal with its own keymap. Merging would require a `mode` switch gating half the body.
- **Inline confirm title renderer into `lua/api/confirm.rs`** — `app/dialogs/confirm.rs` carried just `render_title_into_buf` + `render_title` (one caller: the binding). Move both as private fns; the whole `app/dialogs/` directory retires.
- **`ConfirmPreview` enum drops; notebook preview folds into `smelt.notebook.render`** — once the title renderer stopped reaching for the enum, `from_tool` had one live caller. Diff / FileContent / BashBody variants were unreachable. `app/dialogs/confirm_preview.rs` (298 LOC) deletes.
- **Confirm option-label building → Lua** — `dialogs/confirm.lua` reads `outside_dir` / `approval_patterns` from the `confirm_requested` cell and builds the option list locally. Drops `app/dialogs/confirm.rs::{build_options, cwd_label}`, the parallel `choices` array on `ConfirmEntry`, and the `options` / `cwd_label` fields on the `ConfirmRequested` cell. Same lexicon `confirm_resolved` already publishes.
- **`smelt.reasoning` carved out of `smelt.engine`** — `lua/api/reasoning.rs` mirrors the `smelt.mode` shape. `smelt.engine.reasoning_effort` / `smelt.engine.set_reasoning_effort` retire.
- **`smelt.model` carved out of `smelt.engine`** — `lua/api/model.rs` aligns with the `smelt.model.set(name)` example in ARCHITECTURE.md. `smelt.engine.{model,set_model,models}` retire; `models` → `list`.
- **Session-scoped getters relocate to `smelt.session`** — `smelt.engine.{cost,context_tokens,context_window}` → `smelt.session.*`. The values are session state; the engine namespace ends as a turn-driver / event surface only.
- **`smelt.session.messages` replaces `smelt.engine.history`** — `core.session.messages` is the SoT after P2.a.7b.
- **Dead `smelt.engine.{submit,is_busy}` bindings drop** — both unused by any Lua plugin since the lua/api split.
- **Generalize `:` -> `/` aliasing in dispatcher** — drop hardcoded `:q` / `:qa` matches; any `:`-prefixed input flows through the same dispatch path as `/`-prefixed input.
- **`smelt.clipboard()` `__call` write shim drops** — one Lua consumer (`export.lua`) migrated to `smelt.clipboard.write`. `smelt.clipboard` is now table-only with two explicit verbs.
- **`/reflect` and `/simplify` fully Lua-native** — prompt bodies inline into `plugins/{reflect,simplify}.lua` as Lua strings; the one minijinja conditional becomes a Lua branch on `smelt.engine.multi_agent()`. New `smelt.engine.submit_command(name, body)` binding starts the turn directly. `crates/tui/src/builtin_commands.rs` (89 LOC), `crates/engine/src/prompts/commands/{reflect,simplify}.md`, the `submit_builtin_command` Lua binding, and the `minijinja` dep on `tui` all retire.
- **User-defined `/foo` custom commands fully Lua-native** — file scanning, frontmatter parsing, exec-block evaluation, slash-command registration, and dispatch all move into `runtime/lua/smelt/plugins/custom_commands.lua`. Rust keeps the typed `CustomCommand` / `CommandOverrides` / `RuleOverride` structs that `begin_custom_command_turn` consumes; the Lua plugin fills them via the new `overrides` arg on `smelt.engine.submit_command`. New bindings: `smelt.parse.frontmatter`, `smelt.path.{config_dir,commands_dir}`. New `cmd.register` opt: `queue_when_busy` (defers mid-turn invocation instead of erroring). `custom_commands.rs` shrinks from 322 → 39 LOC; the `InputOutcome::CustomCommand` variant + the three `custom_commands::resolve` callsites + `is_custom_command` / `list` references in completer + events + commands all retire.
- **P4.c statusline composition fully Lua** — `runtime/lua/smelt/status.lua` registers a `core` source via `smelt.statusline.register` whose handler consumes `smelt.statusline.snapshot()` (a single FFI snapshot of working state, throbber spans, vim mode, agent mode, indicators, position, theme colors) and returns the segment list the existing layout engine consumes. Drops the 270-LOC `refresh_status_bar` body; `app/status_bar.rs` collapses to 35 LOC of buffer write. Drops the `compute_status_position` helper, the `StatusPosition` type + its `render` method, and the `StyleState` re-export. New helpers: `super::color_to_ansi(Color)` for projecting palette colors as `u8` for the Lua side. Visibility bumps: `TuiApp::working` (`pub(in app)` → `pub(crate)`), `WorkingState` (`pub(super)` → `pub(crate)`), `WorkingState::{is_animating, is_compacting, spinner_char}`, `focused_overlay_blocks_agent` to expose them to the binding.
- **P4.f cycle-in-Lua** — `runtime/lua/smelt/modes.lua` (bootstrap chunk) overrides `smelt.{mode,reasoning}.cycle` with Lua impls reading new `cycle_list()` bindings (mode falls back to `Mode::ALL` when empty; reasoning returns the configured list verbatim — empty = no-op). Rust `TuiApp::{toggle_mode,cycle_reasoning}` and `protocol::{Mode,ReasoningEffort}::cycle_within` retire; the seven callsites (six `events.rs`, one `lua/api/confirm.rs::_back_tab`) reach Lua via `LuaRuntime::{cycle_mode,cycle_reasoning}`. `_back_tab` splits its single `with_app` block into three sequential phases (gather entry → fire `smelt.mode.cycle` → re-enter `with_app` to inspect decision) so the cycle's reentrant `smelt.mode.set` doesn't nest two `with_app` borrows. Pluggable "register a mode" surface stays out of scope (engine consumes `protocol::Mode` for permission matching).

## Dead-code sweeps

Multiple visibility-hygiene and dead-code passes landed across P4 (demoting
`pub` → `pub(crate)` → private across `ui`, `tui`, `engine`, and `protocol`).
These are mechanical: types behind `pub(crate) mod` still declared `pub` items
that couldn't escape. The dead-code passes also surfaced genuinely unused
code: `ViewportGeom` module, `YankSubst` machinery, named-mark cluster,
`BufType` enum, `modifiable` field on Buffer, `Buffer::wrap_at`, `EngineHandle.permissions`
field, `EngineHandle::notify_agent_message`, `inject_agent_message`/`inject_agent_exited`,
`FocusTarget` enum, `style.rs` module, and dozens of dead pub methods across
content/, input/, completer/, app/, and lua/. Full detail in `git log`.
