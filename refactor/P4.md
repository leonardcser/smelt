# P4 — Lua takes the UX

**Status:** in-progress **Started:** 2026-04-30 **Landed:** —

P4 moves every "what does smelt look/behave like" decision out of `tui`
into `runtime/lua/smelt/`. Rust shrinks to capability provision and
pixel pushing.

Sub-phases (interleave-friendly except for P4.b which is gated on
P1.a-tail `Buffer::attach`):

- **P4.a** — `runtime/lua/smelt/` layout. Create `widgets/`, `dialogs/`,
  `colorschemes/` directories and seed files. Move existing
  `plugins/*.lua` into their target homes (dialogs → `dialogs/`,
  picker recipes → `widgets/`).
- **P4.b** — transcript and diff parsers in Lua via `buf.attach { parser
  = "markdown", on_block = … }`. Gated on `Buffer::attach` (P1.a-tail).
  Drops `transcript_cache.rs`, `transcript_buf.rs`,
  `transcript_present/`, `content/highlight/`, `content/stream_parser`.
  This is also where `tui::parse` lands as the parse-only surface that
  Lua consumes through `smelt.parse` (or the per-language bindings —
  see P3 Decision `9d40423`).
- **P4.c** — reactive statusline. `status.lua` registers a segment spec
  via `smelt.statusline.set({ ... })`; segments bind to cells.
- **P4.d** — dialogs orchestrated in Lua. Each Rust dialog file
  collapses to a request-emit + a resolution primitive on the
  appropriate namespace; Lua composes the panels via
  `smelt.ui.dialog.open` (Overlay + LayoutTree + N Windows).
- **P4.e** — slash commands fully Lua. `runtime/lua/smelt/commands.lua`
  registers every builtin via `smelt.cmd.register`. `RUST_COMMANDS`
  table retires.
- **P4.f** — modes registry in Lua. `modes.lua` exposes `smelt.modes` —
  register a mode (name, display, keymap-overlay).

## Decisions made

- **`prompt_picker.lua` stays separate from `picker.lua`** (`d4bfedc`) — both relocate to `widgets/` as siblings. They share the "select-from-list" shape, but the prompt-docked variant is a non-focusable float that re-filters on every prompt keystroke (Up/Down nav lives in the prompt's vim recipe, not the picker), while the generic picker is a focusable modal with its own keymap. Merging would require a `mode = "modal"|"docked"` switch that gates half the body; the docked variant also routes through `smelt.prompt.open_picker` rather than `smelt.ui.picker.open`. Resolves the open INVENTORY question.
